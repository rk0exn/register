使い方

1

フォーク

GitHub リポジトリをフォークしてください。

2

JSON を追加

domains/your-name.json を作成し、DNS レコードを記述します。

3

PR を送信

Pull Request を作成すると自動バリデーションが走り、マージ後に DNS が反映されます。

対応レコードタイプ

11 種類の DNS レコードに対応しています。

A AAAA CNAME MX TXT NS CAA DS SRV TLSA URL
🛡️ Cloudflare プロキシ
A, AAAA, CNAME では "proxied": true を指定すると Cloudflare プロキシ（CDN・DDoS 保護・SSL）を有効化できます。

ドメイン定義ファイル

{
  "owner": {
    "username": "your-github-name",
    "email": "you@example.com"
  },
  "records": {
    "CNAME": "your-name.github.io"
  }
}
💡 よく使われるレコード例

▸ メールサーバー（MX）

"records": { "MX": [{ "priority": 10, "value": "mail.example.com" }] }

▸ SPF / 検証用（TXT）

"records": { "TXT": ["v=spf1 include:_spf.google.com ~all"] }

ルール

1

1人1サブドメイン
GitHub アカウントにつき1つまで登録可能

a-z

英小文字・数字・ハイフン
2〜63文字。先頭/末尾・連続ハイフン不可

=

PR 送信者 = owner
他人名義の登録は自動で拒否されます

📋

レコードタイプは基本1種類
1つのドメインに設定できるレコードタイプは原則1種類です。ただし NS + DS の組み合わせは許可されます。

⚠

CNAME と A/AAAA は併用不可
CNAME を使用する場合、A や AAAA レコードは同時に設定できません。

無効なファイル名の例

ファイル名理由
My-Site.json大文字を含んでいる
my_site.jsonアンダースコアは使用不可（ハイフンを使う）
-mysite.json先頭がハイフン
mysite-.json末尾がハイフン
my--site.json連続ハイフン
my.site.jsonサブサブドメインは未対応
12345.json数字のみ
a.json短すぎる（2文字以上必要）
www.jsonシステム予約済み
